# tg-bot-image-forwarder

Telegram-бот на Dart, который позволяет сохранять изображения с «именем/фильтром» и затем отправлять нужное изображение по текстовому запросу.

## Как работает

1. В приватном чате бот принимает фото с подписью.
2. Подпись используется как имя фильтра (приводится к нижнему регистру, пробелы убираются).
3. Если фильтр новый — фото сохраняется (fileId и метаданные) в `data/data_storage.json`.
4. Далее бот слушает обычный текст (без команды) и по вхождению имени фильтра в запрос отправляет сохранённое изображение.
5. Есть команды для управления фильтрами и просмотра списка.

## Возможности и команды

- `/start` — Информация о боте.
- `/help` — Список доступных команд.
- `/filter` <filter_name> + фото — Создать/задать фильтр через caption фото (в текущей реализации логика привязана к фото с caption).
- `/filters` — Показать список фильтров порциями по 10 с inline-кнопками (перелистывание поддерживается).
- `/remove` <filter_name> — Удалить фильтр.
- `/edit` <old_filter> <new_filter> — Переименовать фильтр.


### По тексту
Отправка фото выполняется, когда пользователь присылает текст без команды:
- бот ищет, содержит ли текст одно из имен фильтров
- отправляет фото первого/последнего подходящего фильтра


## Структура проекта

- `bin/main.dart` — точка входа: чтение `TOKEN` из `.env` и запуск бота.
- `lib/src/bot.dart` — класс `ImageForwarderBot`: старт/остановка, подключение хранения и обработчиков.
- `lib/src/command/command.dart` — регистрация телекоманд и обработчики команд.
- `lib/src/handler/image_handler.dart` — обработка фото/текста и отправка изображений.
- `lib/src/handler/filter_preview.dart` — UI просмотра списка фильтров (media group + inline keyboard) и обработка callback.
- `lib/src/service/data_storage.dart` — хранение `ImageData` в `data/data_storage.json`.
- `lib/src/service/delete_message.dart` — удаление сообщений из превью списка.
- `lib/src/module/*` — модели данных (`ImageData`, `DeleteMessageData`).

## Конфигурация

### Переменные окружения
В `bin/main.dart` бот ожидает переменную окружения `TOKEN`.
Обычно это задаётся через файл `.env` (подключается библиотекой `dotenv`).

Пример `.env`:

```env
TOKEN=123456:ABC-DEF...
```

## Запуск

```bash
git clone https://github.com/u006E10011/tg-bot-image-forwarder.git
cd tg-bot-image-forwarder
dart pub get
dart run bin/main.dart
```

## Хранение данных

Данные о фильтрах и изображениях сохраняются в:
- `tg-bot-image-forwarder/data/data_storage.json`

Формат:
- ключ — имя фильтра
- значение — `filter`, `fileId`, `fileSize`, `createdAt`.
